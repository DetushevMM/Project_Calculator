# Calculator

Учебный проект на C++17, посвящённый разработке собственного арифметического калькулятора. Проект выполнен поэтапно и включает три независимые части:

1. Tokenize — разбиение входной строки на последовательность токенов.
2. Polish Notation — вычисление выражений, записанных в префиксной (польской) нотации, с использованием абстрактного синтаксического дерева (AST).
3. Calculator — полноценный калькулятор для вычисления арифметических выражений в привычной инфиксной записи с учётом приоритета операций и скобок.

Проект демонстрирует применение современных возможностей языка C++17, рекурсивного нисходящего парсинга (Recursive Descent Parser), объектно-ориентированного программирования и управления памятью с помощью std::unique_ptr.

# Возможности

## Tokenize

Поддерживает токенизацию следующих элементов:

- числа;
- арифметические операции;
- круглые скобки;
- ключевые слова;
- неизвестные идентификаторы.

Поддерживаемые токены:

| Токен | Назначение |
|--------|------------|
| + | сложение |
| - | вычитание |
| * | умножение |
| / | деление |
| % | остаток от деления |
| ^ | возведение в степень |
| ( | открывающая скобка |
| ) | закрывающая скобка |
| sqr | квадрат |
| abs | модуль |
| min | минимум |
| max | максимум |
| число | целочисленная константа |
| неизвестное слово | UnknownToken |

---

## Polish Notation

Поддерживает вычисление выражений, записанных в префиксной записи.

Пример:

+ 3 * 4 5
строится дерево

    +
   / \
  3   *
     / \
    4   5
после чего дерево рекурсивно вычисляется.

Поддерживаемые операции:

- +
- -
- *
- /
- %
- ^
- abs
- sqr
- min
- max
- унарный плюс
- унарный минус

---

## Calculator

Поддерживает обычную математическую запись.

Например,

2 + 3 * (4 - 1)
автоматически преобразуется в дерево выражения

      +
     / \
    2   *
       / \
      3   -
         / \
        4   1
и вычисляется рекурсивно.

# Структура проекта

## Tokenize

Отвечает за разбиение входной строки на последовательность токенов.

Используются:

- std::variant
- std::vector
- std::string_view

---

## Expressions

Содержит классы дерева выражений:

- IExpression
- Constant
- IUnaryOperation
- IBinaryOperation

и конкретные операции

- Plus
- Minus
- Sum
- Subtract
- Multiply
- Divide
- Residual
- Power
- Minimum
- Maximum
- AbsoluteValue
- Square

Каждый узел умеет самостоятельно вычислять своё значение.

---

## Polish Notation

Построение AST для польской записи.

Работа происходит по алгоритму:

Tokenize
        ↓
Построение AST
        ↓
Calculate()
---

## Parser

Рекурсивный нисходящий парсер обычной записи.

Состоит из четырёх уровней.

Expression
        +
        -

↓

Addendum
        *
        /
        %

↓

Power
        ^

↓

Multiplier
        число
        (...)
        +x
        -x
Такой подход автоматически реализует правильный приоритет операций.

---

## Calculator

Главная точка входа.

Выполняет:

1. токенизацию;
2. проверку ошибок;
3. построение дерева;
4. вычисление результата.

# Обработка ошибок

Проект использует два пользовательских исключения.

## UnknownSymbolError

Возникает, если токенизатор обнаружил неизвестный символ.

Пример

text
2 + hello

---

## WrongExpressionError

Возникает при синтаксической ошибке.

Например

text
2 +

(3+4

5*)

2 ** 3
`
